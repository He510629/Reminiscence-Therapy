# 忆往开来·本土化怀旧数字陪伴系统
## 系统设计文档 v1.0

---

## 1. 系统概述

**定位：** 面向轻度认知障碍（MCI）老年人的数字陪伴与早期干预平台，以西安本土怀旧内容为核心，融合AI个性化推荐与多模态交互。

**核心目标：**
- 延缓认知衰退，改善情绪状态
- 老人零门槛上手，单次操作不超过3步
- 支撑 B端机构年费 + C端订阅 + 内容授权多元盈利

---

## 2. 总体架构

系统采用前后端分离、五层架构：

```
┌─────────────────────────────────────────────────┐
│  终端层   平板APP │ 智能电视 │ 音箱 │ 家属手机 │ 机构Web │
├─────────────────────────────────────────────────┤
│  应用层   用户服务 │ 内容服务 │ 游戏服务 │ 陪伴服务 │ 报告服务 │
├─────────────────────────────────────────────────┤
│  AI层    推荐引擎 │ 方言ASR/TTS │ 情绪感知 │ 认知评估  │
├─────────────────────────────────────────────────┤
│  数据层   MySQL │ MongoDB │ Redis │ OSS对象存储    │
├─────────────────────────────────────────────────┤
│  基础设施  云服务器（弹性）│ CDN │ API网关 │ 消息队列  │
└─────────────────────────────────────────────────┘
```

---

## 3. 核心模块

### 3.1 怀旧策展馆

**功能：** 按年代/地域/主题浏览本土化内容，配语音解说，支持家属上传家庭老照片。

**内容数据结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| content_id | UUID | 内容唯一标识 |
| type | Enum | photo / audio / video |
| era | String | 年代标签，如 1960s |
| region | String | 地域标签，如碑林区 |
| theme_tags | Array | 主题标签，如 [老街, 饮食] |
| emotion_tag | Enum | 温馨 / 欢快 / 思乡 |
| cognitive_level | Int 1-3 | 认知刺激强度 |
| copyright_status | Enum | public / licensed / user_uploaded |

---

### 3.2 趣味健脑营

**功能：** 怀旧元素融合认知训练游戏，自适应难度。

| 游戏名称 | 训练认知域 | 怀旧融合 |
|----------|-----------|---------|
| 老物件配对 | 记忆力 | 搪瓷缸、蜂窝煤炉等图片 |
| 跟着戏曲哼旋律 | 语言/音乐记忆 | 秦腔片段跟唱 |
| 老地图连线 | 空间定向 | 上世纪西安城区地图 |
| 方言词语接龙 | 语言能力 | 西安方言语音输入 |
| 节气习俗问答 | 语义记忆 | 西安传统民俗选择题 |

**自适应难度规则：**
- 连续3次正确率 ≥ 80% → 升级难度
- 连续2次正确率 < 50% → 降级难度
- 放弃次数 ≥ 2 → 优先推怀旧内容恢复参与意愿

---

### 3.3 数字陪伴官

**功能：** AI情感对话、生活提醒、情绪感知、家人远程查看。

**对话系统流程：**

```
用户语音输入
    ↓
ASR（支持西安话/关中话）
    ↓
NLU 意图识别（聊天/求助/提醒/回忆）
    ↓
LLM 生成回复（融入西安本土文化知识）
    ↓
TTS 语音播报（支持家人声音复刻）
```

---

### 3.4 认知评估与报告

**采集数据：** 浏览时长、游戏正确率/反应时间、对话频次、情绪分布

**报告规则：**
- **周报**：自动生成，推送至家属手机
- **月报**：含认知域雷达图及变化趋势，B端可查
- **预警**：训练正确率连续下降 >20%，自动通知家属

## 4. 多终端适配

| 终端 | 交互方式 | 核心适配策略 |
|------|---------|------------|
| 平板 | 触屏点击 | 定制启动器，开机自动进入 |
| 智能电视 | 遥控器方向键 | 大画面，遥控导航适配 |
| 智能音箱 | 语音指令 | 全语音，支持方言唤醒 |
| 家属手机 | 标准触屏 | 远程查看、上传素材 |
| 机构Web | PC浏览器 | 数据看板、批量管理 |

**适老化规范（强制）：**
- 最小字号 ≥ 24pt，标题 ≥ 36pt
- 背景/文字对比度 ≥ 7:1
- 所有操作单次点击完成，禁止双击/长按/滑动
- 每个页面提供语音播报按钮
- 误触弹出语音确认，提供显眼返回按钮

## 5. 数据存储

| 数据类型 | 存储介质 | 保留周期 |
|---------|---------|---------|
| 用户信息、画像 | MySQL（加密） | 账户有效期内 |
| 媒体资源（图/音/视频） | OSS 对象存储 | 永久 |
| 行为日志 | MongoDB | 2年 |
| 对话记录（脱敏） | MongoDB（加密） | 1年 |
| 评估报告 | MySQL + OSS | 永久 |
| 热点缓存、Session | Redis | 24小时 TTL |

---

## 6. 安全与隐私

- **传输**：全链路 HTTPS/TLS 1.3，API 使用 JWT 鉴权
- **存储**：用户信息 AES-256 加密，密码 bcrypt 哈希
- **权限**：RBAC 分级（老人 / 家属 / 机构 / 运营 / 管理员）
- **隐私**：最小必要采集；AI训练数据一律去标识化；用户可随时申请删除数据
- **内容版权**：所有内容标注版权状态，授权文件存档备查

## 7. 主要接口

```
POST /api/v1/auth/login            # 用户登录
GET  /api/v1/user/profile          # 获取用户画像
GET  /api/v1/content/recommend     # 个性化内容推荐
GET  /api/v1/content/{id}          # 内容详情
POST /api/v1/content/upload        # 上传家庭私有内容
GET  /api/v1/game/daily            # 今日推荐训练
POST /api/v1/game/result           # 提交游戏结果
POST /api/v1/chat/send             # 发送对话
GET  /api/v1/report/weekly         # 获取周报
GET  /api/v1/institution/dashboard # 机构看板
```

统一响应格式：`{ "code": 200, "message": "success", "data": {...} }`

## 9. 开发路线图

**阶段一（0-6个月）MVP**
- 怀旧策展馆基础版 + 3款健脑游戏 + 文字AI对话
- 平板端APP + 适老化测试
- 西安本土内容库初始500条
- 2-3家机构免费试点

**阶段二（7-18个月）标准版**
- 方言语音识别 + AI语音对话
- 电视端、音箱端适配
- 家属手机端 + 机构B端看板上线
- 内容库扩展至2000条，开启付费合作

**阶段三（19-36个月）专业版**
- 认知评估报告 + 长护险机构对接
- 高级订阅功能（声音复刻、家族记忆拼贴）
- 向陕西省内其他城市拓展本地化内容
